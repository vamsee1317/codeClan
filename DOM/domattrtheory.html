DOM Attribute Manipulation – Complete Theory with Examples

DOM attribute manipulation allows JavaScript to read, write, update, and verify the attributes of HTML elements in real-time. These operations are heavily used in modern frontend development, especially in eCommerce, dashboards, admin UI, form handling, and interactive components.

1. setAttribute(attribute, value)

Definition: Adds a new attribute or updates an existing one.

Syntax:

element.setAttribute("attributeName", "value")

Real-time Use Cases:

  • Disable buttons on form submission
  • Lazy loading images by setting src dynamically
  • Adding custom data-* attributes for backend or analytics
  • Changing theme using data-theme="dark"

Example:

const btn = document.getElementById("buyBtn");
btn.setAttribute("disabled", true);

This is how real eCommerce apps disable the Buy button when item is unavailable.

2. getAttribute(attribute)

Definition: Retrieves the value of an attribute.

Syntax:

element.getAttribute("attributeName")

Real-time Use Cases:

  • Getting product ID (data-id) before sending API calls
  • Reading href of a link before redirecting
  • Reading current theme (data-theme)

Example:

const product = document.querySelector(".product-card");
const id = product.getAttribute("data-id");
console.log("Product ID:", id);

3. hasAttribute(attribute)

Definition: Checks if an attribute exists on an element.

Syntax:

element.hasAttribute("attributeName")

Real-time Use Cases:

  • Check if a button is already disabled
  • Check if a product card has data-out-of-stock
  • Check if an image has a src assigned
  • Check ARIA attributes for accessibility

Example:

if (!btn.hasAttribute("disabled")) {
    btn.setAttribute("disabled", true);
    console.log("Button disabled.");
}

4. Class Attribute Manipulation

Class manipulation is done using element.classList, not setAttribute. This is cleaner, easier, and used throughout production applications.

Available classList methods:

  • add("class") – Adds a class
  • remove("class") – Removes a class
  • toggle("class") – Adds if missing, removes if present
  • contains("class") – Checks if class exists

Real-time Use Cases:

  • Toggling wishlist heart icon
  • Showing/hiding mobile menus
  • Highlighting active navbar items
  • Dark/light theme toggles
  • Adding error/success classes in forms

Example:

const card = document.querySelector(".card");

// Toggle highlight
card.classList.toggle("highlight");

// Check if highlight exists
if (card.classList.contains("highlight")) {
    console.log("Product is highlighted");
}

Complete Mini Example (All Methods Together)

This example simulates a real production environment used in eCommerce.

<div class="product-card" id="card" data-id="555">
    <button id="wish">Add to Wishlist</button>
</div>

<script>
const card = document.getElementById("card");
const wish = document.getElementById("wish");

// setAttribute
wish.addEventListener("click", () => {
    // Toggle wishlist UI class
    wish.classList.toggle("active");

    // Add custom attribute
    if (wish.classList.contains("active")) {
        card.setAttribute("data-wishlist", "true");
    } else {
        card.setAttribute("data-wishlist", "false");
    }
});

// getAttribute
console.log("Product ID:", card.getAttribute("data-id"));

// hasAttribute
if (!card.hasAttribute("data-out-of-stock")) {
    console.log("Product is currently available.");
}
</script>

Summary

  • setAttribute() – Add/update attributes
  • getAttribute() – Read attributes
  • hasAttribute() – Check existence of attributes
  • classList – Clean way to modify classes

These methods are essential for building interactive UI in production apps like Amazon, Flipkart, admin dashboards, LMS platforms, booking systems, and more.
